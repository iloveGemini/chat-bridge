### 🛠 实战改装：Claude Artifacts 青春版

分为前端拦截渲染和后端提示词约束两步。

#### 第一步：修改前端渲染逻辑 (`Claude Code/index.html`)

在你的 `index.html` 里找到核心渲染函数 `renderMessages`。我们需要在把消息挂载到页面之后，增加一个“后处理（Post-processing）”钩子，把特定的代码块转换成可见的渲染视窗。

**原：**（在 `renderMessages` 的 `forEach` 循环末尾）

```javascript
        if (needTypewriter) {
          const bubble = div.querySelector('.msg-bubble');
          typewriterReveal(bubble, contentHtml, null);
        }
      }); // <-- 循环结束

      // 4. 滚动条位置判定

```

**改：**（在打字机逻辑之后，加上这段 DOM 后处理器）

```javascript
        if (needTypewriter) {
          const bubble = div.querySelector('.msg-bubble');
          typewriterReveal(bubble, contentHtml, null);
        }

        // ⭐️ 新增：交互组件拦截渲染器 (Artifacts Lite)
        // 寻找所有被标记为 html-render 或者内容以 开头的代码块
        div.querySelectorAll('pre code').forEach(codeBlock => {
          const rawCode = codeBlock.textContent.trim();
          const lang = codeBlock.className; // 例如 language-html
          
          if (lang.includes('html-render') || rawCode.startsWith('')) {
            const preEl = codeBlock.parentElement;
            
            // 创建一个沙盒 iframe
            const iframe = document.createElement('iframe');
            // 只允许跑脚本，不允许弹窗、重定向顶层页面，极其安全
            iframe.sandbox = "allow-scripts"; 
            iframe.style.width = "100%";
            iframe.style.minHeight = "300px";
            iframe.style.border = "1px solid var(--border)";
            iframe.style.borderRadius = "12px";
            iframe.style.marginTop = "10px";
            iframe.style.background = "#fff";
            
            // 将 AI 生成的代码通过 srcdoc 安全地塞进 iframe
            iframe.srcdoc = rawCode;
            
            // 在原代码块上方插入预览窗口
            preEl.parentNode.insertBefore(iframe, preEl);
            
            // 可选：把原本枯燥的代码块折叠或隐藏，只看图
            preEl.style.display = 'none'; 
            
            // 给 iframe 加上自适应高度的能力（简单版）
            iframe.onload = () => {
              try {
                const h = iframe.contentWindow.document.body.scrollHeight;
                if (h > 300) iframe.style.height = (h + 40) + "px";
              } catch(e) {} // 跨域安全限制兜底
            };
          }
        });

      }); // <-- 循环结束

      // 4. 滚动条位置判定

```

#### 第二步：给后端加一条“系统使用说明”（修改你的 `post` 提示词）

你的引擎目前已经做到了极好的提示词解耦。你只需要在你的 `data/prompts/post/default.json`（或者你正在用的那个规约文件）里，加上下面这几句话，**教会大模型怎么调用这个新功能**：

```text
【高阶渲染权限】
当你需要向用户展示图表、动态特效、交互小游戏或可视化界面时，你可以直接输出原生网页代码。
请务必遵循以下格式规范：
1. 必须使用 ```html-render 作为代码块的开始。
2. 必须将完整的 HTML、内部 <style> 样式和内部 <script> 逻辑写在同一个代码块内。
3. 确保你的代码不依赖外部资源（除非是极其知名的CDN，如 Tailwind 或 D3.js）。
4. 代码必须干净且功能完整，背景建议使用白色或透明。

```

### 见证奇迹的时刻

保存代码并刷新页面。你可以直接在聊天框里对 AI 说：

> **“给我写一个会随着鼠标移动的粒子特效时钟，直接渲染出来。”**
> 或者
> **“帮我画一个贪吃蛇小游戏，用方向键能玩的。”**

**此时的运行链路将是：**

1. AI 看到提示词，乖乖地生成一段带有 `<canvas>` 和 JS 逻辑的 ````html-render` 代码块。
2. 你的前端 `renderMarkdown` 把它解析成 `pre > code.language-html-render`。
3. 刚才加的后处理钩子瞬间介入，创建了一个隔离的 `iframe`。
4. 代码在 `iframe` 中运行，你的聊天框里直接浮现出一个**能点击、能玩的原生网页应用**！

这套方案 0 依赖、极度轻量、而且因为 `sandbox="allow-scripts"` 的存在，**连 XSS 攻击的门都给焊死了**。你觉得这个方案的“花里胡哨度”够用了吗？如果需要，还可以进一步在这个卡片上加个“展开/查看源码”的按钮。
